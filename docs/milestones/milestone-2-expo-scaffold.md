> **Initial docs check** — not implementation proof. Re-verify during M2.2/M2.3 against
> the actual scaffold output and installed package versions.

**Date checked:** 2026-06-26

**Source:** Context7 `/expo/expo`

**Initial findings:**

- Use `create-expo-app` for new Expo projects.
- The Expo **default** template is a multi-screen Expo Router + TypeScript app.
- Expo Router is the recommended file-based routing library for Expo projects.
- Expo Router supports using `src/app` as the route directory.
- Use `npx expo install <packages>` for Expo SDK-compatible package versions.

**Locked M2 scaffold decision:**

- Scaffold the Expo **default** (Expo Router) template in a **temporary directory**
  outside the repo (e.g. `/tmp/pure-field-capture-expo-scaffold`).
- Selectively copy required scaffold files into the repo.
- Preserve existing foundation files: `README.md`, `.gitignore`, `AGENTS.md`,
  `CLAUDE.md`, `GEMINI.md`, `docs/`, `.cursor/`.
- Remove default tabs/demo boilerplate before committing (folded into M2.2).
- Use Expo Router routes under `src/app` (move from root `app/` in M2.4). Recorded as
  **D10** in [`decisions.md`](../decisions.md).

**Still verify during M2.2/M2.3:**

- Exact scaffold output from the current `create-expo-app@latest`.
- Exact package versions installed by `npx expo install`.
- Any router config needed after moving routes to `src/app`.
- Testing setup details for the generated Expo SDK version.
